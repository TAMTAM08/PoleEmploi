Programme de la formation

  • Développement
    • Back-end

      Le back-end est la partie qui gère la requête reçue et génère la réponse renvoyée (une page HTML dans le cas d'un site public) à partir de la logique interne et de la base de donnée.

      • Base de donnée
        • + - Conceptions, norme

          • Il existe un certain nombre de normes en conception de base de donnée qui ont pour objectif notamment : - de faciliter les traitements automatisés - d'éviter la redondance de donnée - d'assurer la cohérence des données

        • + - SQL
          • Structured Query Langage : le langage usuel de requête en base de donnée
        • Différents SGBDR
      • Java
        • API, structure, concepts
        • + - Programmation Orientée Objet
          • Le concept n'est pas limitée à Java, mais particulièrement présent dans le langage
        • + - JPA / Hibernate
          • Le standard d'accès aux données en java et son implémentation la plus utilisée
    • Front-end

      Le front-end est la partie vue par l'utilisateur.

      • + - HTML / template

        • Le langage de base des pages web, qui sert de support à tous les éléments de la page (image, script, css, etc ...) Il est représenté dans la base de donnée sous forme de template génériques qui sont remplis avec les informations spécifiques.

      • + - Css
        • Cascading Style Sheet : les feuilles de style, qui contiennent les informations de mise en forme.
      • Javascript

        Le presque seul langage de programmation dynamique exécutable dans le navigateur, il permet de modifier dynamiquement et rapidement le code HTML et donc l'affichage de la page.

        • + - AngularJS
          • Un framework (ensemble de librairie et de fonction) de développement en javascript
    • Transverse
      • Tests Unitaires
      • + - Algorithmie

        • Principes de base d'algorithmie, notamment la question des accès aux données, centrales en informatique (liée aux structures de donnée java, aux index en base de donnée, etc ...)

  • Outils
    • Outils de gestion de projet
      • + - Maven

        • Outil de gestion de dépendance : on indique dans un fichier les dépendances du projet et maven gère la configuration avec les différents programmes : apache, eclipse

      • + - Git

        • Outil de gestion de configuration : on y sauvegarde les fichiers (push) afin de les partager et de pouvoir retrouver les différentes versions (et modification sur chacun)

      • Outils de débogage
      • + - Jenkins

        • Outil d'intégration continue : a chaque modification du code, on lance automatiquement les tests et/ou l'intégration, afin de détecter les bugs au fur et à mesure

    • Outil de conception
      • + - UML

        • Universal Markup Langage : comme son nom l'indique on peut l'utiliser pour représenter beaucoup de diagramme différents, mais en ici principalement les modèles de donnée et les diagramme de classe

  • Architecture
    • + - Architecture MVC

      • Model View Controller : il s'agit de l'architecture usuelle côté serveur, où l'on sépare les 3 aspects ; Modèle de donnée, View (affichage) et Contrôlleur (gestion des requêtes)

    • Architecture N-tier
    • Architecture REST
